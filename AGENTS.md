# Agent Instructions

## Project Knowledge

`docs/solutions/` contains documented solutions to past problems and workflow lessons, organized by category with YAML frontmatter such as `module`, `problem_type`, and `tags`. It is relevant when implementing, debugging, reviewing, or making decisions in documented areas.

## Final Review

For non-trivial implementation work, the final verification/review stage includes a compound pass: capture reusable mistakes, lessons, or process improvements in `docs/solutions/` so the next execution-review cycle does not repeat them.
